# Changelog

## 2026-06-14

- Removed stray artifact directory `.codex-run/` (run logs + screenshots, ~16MB) and added `.codex-run/` to `.gitignore`.
- Removed stray `Claude.reg` (Windows registry export containing a plaintext API key) from the repo root; added `*.reg` to `.gitignore`. Registry settings already applied under HKCU are unaffected.
- Backend: added missing endpoints (delete post/trip/comment, edit comment) + list pagination (user posts/saves, comments, points inbox); hardened robustness (feed-impression logging instead of swallowing, AMap HTTP client timeouts, media size/MIME validation, PBKDF2 password hashing with transparent legacy SHA-256 fallback); demo seed now attaches real per-point photo URLs for the gallery↔map demo.
- Web: full redesign into a centered phone-frame, Instagram-style UI (new globals.css design system, bottom tab bar, feed, post detail with photo↔map sync, studio/record/profile/favorites/user/auth pages); removed legacy desktop components and unused logic modules.
- Added `deploy-backend.ps1` (one-command build → upload → backup → swap → restart → verify).
- Deployed the updated backend jar to Aliyun ECS (`tripin-api` container); verified by md5 match + health check. No DB migration needed (schema unchanged); env vars / AMap key / uploads untouched.
- Web: backend API base URL is now configurable via `NEXT_PUBLIC_API_BASE_URL`, centralized in `src/lib/config.ts` (api client + auth route handlers all reference it). Added `apps/web/.env.example` (template, committed) and `.env.local` (gitignored). Same codebase switches between local (`localhost:3001`) and server (Aliyun) backend by editing one line. Verified web→server works for both SSR and browser (CORS) requests.
- Web: home feed cards now show an in-feed swipeable photo gallery (pointer/touch swipe + arrows + dots) that two-way-syncs with the route mini-map points (swiping into a stop's photo highlights that map point; tapping a map point jumps the gallery to that stop's first photo). Extracted a shared `useGalleryMapSync` hook + `GalleryViewer` (reused by the post detail page). Bottom tab bar reworked to 5 slots with a raised center 工作台 button + 收藏 tab. Broad visual de-flattening (layered blue-tinted shadow tokens, glassy gradient top/tab bars, elevated mini-map block, gradient accents + micro-interactions).
- Web: applied 10 adversarial-review fixes — gallery swipe uses setPointerCapture + primary-pointer guard + pointercancel (fixes off-edge swallowed swipes and right-click/multi-touch false navigation); `getFeed` per-post points enrichment now runs with bounded concurrency (batches of 5) instead of unbounded SSR fan-out; accessibility added (gallery counter `aria-live`, dots `aria-hidden`, like/save `aria-pressed` + state-driven `aria-label`, keyboard support on stop rows / thumbnails / map markers with meaningful alt, SVG `role`/`aria-label`); detail-page mini-map indent aligned with the feed.
- Backend + Web: user-uploaded photos are now stored on and served by the backend instead of per-machine local disk. Added `UploadConfig` (static handler serving `UPLOADS_DIR` at `/api/uploads/**`) + `POST /v1/media/upload` (multipart, 20MB limit). Web `/api/uploads` route forwards the file to the backend; `mediaUrl`/`avatarUrlOf` resolve `/uploads/*` against `API_ORIGIN` (`${origin}/api/uploads/*`). Fixes "images not showing" (files previously lived only on whichever machine performed the upload) — existing server images now load from any device/deployment. Backend redeployed.
- Web: interactive non-native elements (map markers / stop rows / thumbnails) no longer show the browser's mouse-focus outline box; keyboard focus ring kept via `:focus-visible`.
- Web: color system reworked to a restrained, premium palette ("高级克制") — warm-neutral grounds (`#efece7`/`#f6f4f1`), low-saturation slate-blue accent (`#3f5d86`), neutral soft shadows, toned-down gradients and background washes, muted route-line/marker colors in RouteSketch.
- Tooling: `deploy-backend.ps1` rewritten ASCII-only (the previous Chinese comments mis-decoded under a Chinese-locale Windows PowerShell and broke the script).
- Web Studio: real AMap (高德地图) integration. Added a JS-API loader (`src/lib/amap.ts`) + `StudioMap` component that renders an interactive AMap (numbered markers, route polyline, click-to-add, marker drag, point focus) when `NEXT_PUBLIC_AMAP_JS_KEY` is set, and gracefully falls back to the route sketch otherwise. Keys read from env (mirrors mobile `EXPO_PUBLIC_AMAP_JS_KEY`/`SECURITY_CODE`); left blank because none exist in the repo yet — set the env var to enable the live map.
- Web Studio: major UX rebuild for easier route building — search a place and tap to add it instantly (AMap POI search via backend), add by current location or by clicking the map (reverse-geocoded), tap a point to focus it on the map, drag markers to adjust position, reorder / edit (with per-point photos) / delete, then publish. Replaces the old always-open multi-field form.
- Backend: fixed TripPoint reorder/reindex violating the `(tripId, sequence)` unique constraint mid-update — now a two-phase renumber (temporary negatives → final sequence). Redeployed.
- Web: palette reworked to a "polished steel blue" metallic gradient (per request) — accent `#34618f` + steel anchor ramp; multi-layer metallic gradients (diagonal gloss highlight + vertical light→dark body) on CTA buttons, the raised center tab button, and avatar rings, with inset top highlight / dark inner edge / depth shadows and a pressed-in `:active`; metallic brand-text gradient. Also swept the remaining hardcoded bright-blues (input/search focus rings, stop-row active, end badge, current thumbnail, raised-tab ring) to steel-blue. Increased feed post spacing (18px/22px padding + a 6px warm-gray divider between posts).
